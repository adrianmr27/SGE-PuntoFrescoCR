1. Como administrador quiero un formulario para crear roles dentro del sistema para definir los distintos niveles de acceso de los usuarios.

Escenario 1: Crear un rol

En caso que el administrador acceda al formulario de roles cuando ingrese el nombre del rol y presione guardar el sistema registra el nuevo rol

Escenario 2: Validación de duplicado

En caso que el administrador intente crear un rol con un nombre que ya existe cuando ingresa el nombre del rol y ya existe un rol con ese nombre el sistema muestra un mensaje de error indicando que el nombre del rol ya está en uso.

Escenario 3: Validar datos

En caso de datos incompletos cuando se intenta guardar el sistema muestra errores

2. Como administrador quiero visualizar y editar o desactivar roles existentes para ajustar la estructura de permisos del sistema.

Escenario 1: Editar rol

En caso que exista un rol registrado cuando el administrador modifique su información el sistema actualiza el rol

Escenario 2: Desactivar rol

En caso que exista un rol activo cuando el administrador cambie su estado a inactivo el sistema desactiva el rol

Escenario 3: Visualizar roles

En caso que existan roles cuando el administrador accede el sistema muestra la lista

3. Como administrador quiero asignar permisos a cada rol para controlar el acceso a los distintos módulos y funcionalidades del sistema.

Escenario 1:Asignar permisos a rol

En caso que exista un rol registrado cuando el administrador seleccione los permisos correspondientes y guarde el sistema asigna los permisos al rol

Escenario 2: Modificar permisos de rol

En caso que un rol tenga permisos asignados cuando el administrador cambie los permisos el sistema actualiza los permisos del rol

Escenario 3: Validar asignación

En caso que el administrador necesite revisar qué permisos tiene asignado un rol específico cuando el administrador selecciona un rol y accede a la pestaña o sección de "Permisos" el sistema muestra qué módulos y funcionalidades tiene habilitados actualmente ese rol.

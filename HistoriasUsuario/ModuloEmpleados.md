1. Como administrador quiero un formulario para registrar, editar y activar o desactivar empleados con datos como nombre completo, identificación, puesto, teléfono, correo, dirección, contacto de emergencia, alergia a medicamentos y estado (Activo/Inactivo) para mantener actualizada la información del personal de la empresa.

Escenario 1: Registrar empleado

En caso que el administrador acceda al formulario de empleados cuando ingrese los datos y presione guardar el sistema registra el empleado

Escenario 2: Editar empleado

En caso que exista un empleado registrado cuando el administrador modifique los datos el sistema actualiza la información del empleado

Escenario 3: Activar/Inactivar empleado

En caso que existan empleados registrados cuando el administrador cambia estado el sistema actualiza el estado del usuario alterado

2. Como administrador quiero ver y consultar toda la información personal de los empleados, así como visualizar la distribución del personal por área o puesto para apoyar decisiones administrativas.

Escenario 1: Visualizar información consolidada

En caso de empleados registrados cuando el administrador accede a la vista de lista de empleados el sistema despliega la información completa de los empleados al hacerles clic

Escenario 2: Visualizar distribución por área o puesto

En caso de múltiples empleados con diferentes puestos cuando el administrador accede a la sección de distribución el sistema muestra la distribución del personal agrupada

Escenario 3: Filtrar información de empleados

En caso de lista de empleados cuando el administrador aplica filtros por área o puesto el sistema muestra los resultados filtrados

3. Como empleado quiero poder visualizar y editar mi información personal como padecimientos o enfermedades, contactos de emergencia, teléfono y correo.

Escenario 1: Ver información personal

En caso de usuario autenticado cuando accede a su perfil el sistema muestra sus datos

Escenario 2: Editar información personal

En caso de actualización cuando modifica datos el sistema guarda cambios

Escenario 3: Restricción de edición

En caso que el empleado intente modificar campos que no debería editar (nombre, identificación, puesto) cuando el empleado hace clic sobre esos campos el sistema mantiene los campos deshabilitados (grises) o no permite la edición, mostrando posiblemente un tooltip que indique "Solo editable por administrador".
